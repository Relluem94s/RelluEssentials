# Documentation

## `public class ConfigHelper`

> ⚠️ **Deprecated since version 4.3.3** – This class is for **legacy migration purposes only**
> and will not receive further development. It exists solely to assist in migrating
> old plugin versions from config files to the database. Do not use this class in
> normal operation with newer versions.
>
> This class is annotated with `@ApiStatus.Internal` and is not part of the public API.

- **Author:** rellu

---

## Constructor

### `public ConfigHelper(String name)`

Creates a new `ConfigHelper` instance for the given config file name.
If the file does not exist, `configFound` is set to `false` and no configuration is loaded.

| Parameter | Type | Description |
|---|---|---|
| `name` | `String` | The name of the config file (without `.yml` extension) |

---

## Fields

### `configFound`

- **Type:** `boolean`
- **Default:** `true`
- **Description:** Indicates whether the config file was found on disk.
  Set to `false` if the file does not exist at construction time.

---

## Methods

### `public List<PlayerEntry> getPlayers()`

Reads all player entries from the `player` section of the config file.

Logs each found player to the console via `consoleSendMessage`.

- **Returns:** `List<PlayerEntry>` – All players found in the config file
- **Throws:** `NullPointerException` – If the `player` config section is missing

---

### `public List<LocationEntry> getHomes(PlayerEntry p)`

Reads all home locations for the given player from the config file.

The home type is determined by name: `"death"` is mapped to type `2`, all others to type `1`.
Logs each found home to the console via `consoleSendMessage`.

| Parameter | Type | Description |
|---|---|---|
| `p` | `PlayerEntry` | The player whose homes should be retrieved |

- **Returns:** `List<LocationEntry>` – All home locations for the given player
- **Throws:** `NullPointerException` – If the player's home section is missing
